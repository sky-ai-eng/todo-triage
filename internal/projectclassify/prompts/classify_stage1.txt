You are scoring how well a single work item (a GitHub PR or a Jira issue) belongs to a specific project. The project is defined by its name, its description (if any), and an excerpt of its curated knowledge-base files (if any). Your job is to return ONE number between 0 and 100 — your confidence that this entity belongs to this project.

# Calibration

Score 0-100 reflects your **confidence** that this entity belongs to this project. Lack of information is a reason to score LOW, not a reason to guess. If the project context is sparse (short description, no knowledge base) and the entity title/body don't explicitly reference what little context you have, return a score below 30. Do NOT round up because you "might be wrong about it being unrelated."

A score of 60+ means: "I am confident this entity is part of this project's scope." Below that the entity stays unassigned. Many projects vote in parallel — only the project that genuinely matches should score high.

Project descriptions are often the strongest signal — short, focused descriptions usually express the user's intent precisely. A 5-line description of "Auth migration: replace session storage with new JWT scheme" should score high on a PR titled "Migrate session token validation" even if there is no knowledge base. Conversely a project named "Misc work" with no description should score nearly everything LOW because there is no signal to anchor on.
